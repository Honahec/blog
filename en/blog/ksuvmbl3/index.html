Deploying VuePress Projects with GitHub Actions and GitHub Pages

About 590 wordsAbout 2 min

ReprintDeploymentFrontend

2025-10-10

Configuring GitHub Actions

The first step is to build the project through GitHub workflows.

Below is the configuration file provided by vuepress-theme-hope, which should work for any VuePress project:

Additional note to the original: Vitepress also (almost) works

name: 部署文档

on:
  push:
    branches:
      - main

permissions:
  contents: write

jobs:
  deploy-gh-pages:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
          # 如果你文档需要 Git 子模块,取消注释下一行
          # submodules: true

      - name: 设置 pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm

      - name: 安装依赖
        run: |
          corepack enable
          pnpm install --frozen-lockfile

      - name: 构建文档
        env:
          NODE_OPTIONS: --max_old_space_size=8192
        run: |-
          pnpm run build
          > src/.vuepress/dist/.nojekyll

      - name: 部署文档
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          # 部署文档
          branch: gh-pages
          folder: src/.vuepress/dist

This action triggers on every push to the main branch, builds the VuePress project, and pushes the generated static files to the gh-pages branch.

When using this, you need to modify the following:

  • branches: If your main branch isn't main, change it to your main branch name.
  • Choose the node and pnpm versions according to your project needs. If you don't use pnpm, just remove the related content.
  • folder: In many cases, the source file directory might not be src, but docs or blog, etc.

Configuring GitHub Pages

I have to say GitHub Pages documentation is poorly written here, it doesn't even clearly explain how to configure CNAME records for custom domains.

If you need to use a custom domain, it's recommended to first set up CNAME resolution for the domain you want to use, pointing to your-github-username.github.io.

Then, in your GitHub repository settings, find the "Pages" tab.

Branch should be set to the gh-pages branch generated in the previous step, and the directory should be set to root / (root).

If you need a custom domain, you can also fill it in on this interface.

GitHub can automatically generate an SSL certificate for your custom domain after verifying the DNS records. This step is usually quick, but it may still get stuck for several hours when first added. There's currently no known effective solution, you can only wait. If DNS information verification fails for a long time, you can try removing the custom domain and adding it again.

Custom Domain Becomes Invalid After Redeployment

GitHub stores custom domains by placing a file named CNAME in the root directory of the selected Git branch, with the file content being the custom domain.

In the above setup, each time you redeploy, GitHub Actions replaces the content of the gh-pages branch with the newly generated static files, simultaneously deleting the CNAME file.

To preserve this file after each deployment, the simplest method is to manually create a CNAME file in the .vuepress/public directory. You can check the file automatically generated by GitHub in gh-pages for the specific content.

The following is a supplement to the original article

You can optionally auto-generate the CNAME file in actions

Just add this to the commands run when building the documentation:

echo "yourdomain.com" > src/.vuepress/dist/CNAME
