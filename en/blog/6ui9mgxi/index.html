Introduction to Bitmask DP

About 1738 wordsAbout 6 min

Algorithm

2022-07-06

Definition

Bitmask Dynamic Programming, abbreviated as Bitmask DP, uses binary representation to compress states into integers to optimize state transitions.

A Simple Example

For a 0-1 knapsack problem (everyone knows this, right?)

There are n≤15n\leq 15n≤15 items and a knapsack with unlimited capacity. The iii-th item occupies space wiw_iwi​ and has value viv_ivi​. Find the value and space occupied for each possible selection.

1. Define the State

We need to find the value for each selection, and for a 0-1 knapsack, each item has only two states: selected or not selected.

So the state we define should be the selection status of these nnn items.

We could easily think of using an nnn-dimensional array, where if the iii-th dimension's index is 111, it means selecting the (i+1)(i+1)(i+1)-th item, otherwise not selecting it.

Assuming we have 555 items, then dp[1][0][1][1][0]dp[1][0][1][1][0]dp[1][0][1][1][0] represents the value obtained by selecting items 1,3,41, 3, 41,3,4.

But would you really want to declare dimensions one by one? ()

We realize that for this problem, we can use binary to represent all cases, and if we convert it to decimal, we can use 111 dimension to express the information that previously required nnn dimensions.

For example, we can obviously use 00000∼1111100000\thicksim 1111100000∼11111 to represent all cases, which converts to decimal as 0∼(1<<5−1)0\thicksim (1<<5-1)0∼(1<<5−1).

2. How to Transition

A "selected" state can only transition from a "not selected" state.

For example, dp[10000(2)]=dp[00000(2)]+v[1]dp[10000_{(2)}]=dp[00000_{(2)}]+v[1]dp[10000(2)​]=dp[00000(2)​]+v[1], dp[11000(2)]=dp[10000(2)]+v[2]=dp[01000(2)]+v[1]dp[11000_{(2)}]=dp[10000_{(2)}]+v[2]=dp[01000_{(2)}]+v[1]dp[11000(2)​]=dp[10000(2)​]+v[2]=dp[01000(2)​]+v[1]

3. Code

#include<cstdio>

int dp[2][(1 << 15) + 5]; //dp[0] for value, dp[1] for space
int n, w[20], v[20];

void print(int s);

int main() {
	scanf("%d", &n);
	for(int i = 0; i < n; ++i) scanf("%d%d", &w[i], &v[i]);
	for(int i = 0; i < (1 << n); ++i) {
		for(int j = 0; j < n; ++j) {
			if(!(i & (1 << j))) {
				dp[0][i | (1 << j)] = dp[0][i] + v[j];
				dp[1][i | (1 << j)] = dp[1][i] + w[j];
			}
		}
	}
	for(int i = 0; i < (1 << n); ++i) {
		print(i);
		printf("\t%d\t%d\n", dp[0][i], dp[1][i]);
	}
	return 0;
}

void print(int s) {
	if(!s) printf("0");
	int k = 0;
	for(; (1 << k) <= s; ++k) {
		if(s & (1 << k)) printf("1");
		else printf("0");
	}
}

Non-attacking Kings

Place KKK kings on an N×NN\times NN×N chessboard so they don't attack each other. How many placement schemes are there?

A king can attack one square in each of the eight directions: up, down, left, right, upper-left, lower-left, upper-right, and lower-right, for a total of 888 squares.

Solution

We define dp[i][j][l]dp[i][j][l]dp[i][j][l] to represent the number of valid schemes for the first iii rows, where row iii has state jjj, and lll kings have been placed on the board.

For each state jjj, if a binary bit is 111, it means a king is placed at that position, otherwise not.

For example, in the figure below, j=101001(2)j=101001_{(2)}j=101001(2)​, sta[j]=3sta[j]=3sta[j]=3

Let the current row state be jjj and the previous row state be kkk. Under the premise of ensuring the current row and previous row don't conflict, enumerate all possible kkk for transition, giving the transition equation:

f[i][j][l]=∑f[i−1][k][l−bitcount(j)]f[i][j][l]=\sum f[i-1][k][l-bitcount(j)] f[i][j][l]=∑f[i−1][k][l−bitcount(j)]

Code

#include<cstdio>

int n, p;
long long dp[11][(1 << 9) + 5][101], ans;
int bitcount[(1 << 9) + 5];

int main() {
	scanf("%d%d", &n, &p);
	dp[0][0][0] = 1;
	for(int i = 0; i < (1 << n); ++i) {
		int t = i;
		while(t) {
			if(t & 1) ++bitcount[i];
			t >>= 1;
		}
	}
	for(int i = 1; i <= n; ++i) {
		for(int j = 0; j < (1 << n); ++j) {
			if(!(j & (j << 1 | j >> 1))) {
				for(int k = 0; k < (1 << n); ++k) {
					if(!(j & (k | k << 1 | k >> 1))) {
						for(int l = bitcount[k]; l <= p; ++l) {
							dp[i][j][l] += dp[i - 1][k][l - bitcount[j]];
						}
					}
				}
			}
		}
	}
	for(int i = 0; i < (1 << n); ++i) ans += dp[n][i][p];
	printf("%lld\n", ans);
	return 0;
}

Artillery Positions

The generals at headquarters plan to deploy their artillery units on an N×MN\times MN×M grid map.

An N×MN\times MN×M map consists of NNN rows and MMM columns. Each cell on the map can be either mountain (represented by H\texttt{H}H) or plain (represented by P\texttt{P}P), as shown below.

At most one artillery unit can be deployed on each plain cell (artillery units cannot be deployed on mountains). An artillery unit's attack range on the map is shown as the black area in the figure:

If an artillery unit is deployed on the plain marked in gray on the map, the black grids represent the area it can attack: two cells left and right horizontally, and two cells up and down vertically.

Other white grids on the map cannot be attacked. As can be seen from the figure, the artillery's attack range is not affected by terrain.

Now, the generals are planning how to deploy artillery units. Under the premise of preventing friendly fire (ensuring that no two artillery units can attack each other, i.e., no artillery unit is within the attack range of another), what is the maximum number of our artillery units that can be placed in the entire map area?

Solution

We define dp[l][s][i]dp[l][s][i]dp[l][s][i] to represent the maximum number of artillery units that can be placed when the previous state is lll, the current state is sss, and we've considered up to row iii. bin[i]bin[i]bin[i] represents the terrain of row iii.

Then we realize that for each row iii, we only need rows i,i−1,i−2i, i-1, i-2i,i−1,i−2, so we consider using a rolling array for optimization (otherwise MLE  0MLE\ \ 0MLE  0).

Same as non-attacking kings, for each state lll (or sss), if a binary bit is 111, it means placing artillery, otherwise not.

Let the state two rows above be jjj. Under the premise of satisfying the problem requirements, we have the transition equation:

dp[l][s][i]=max(dp[j][l][(i−1)])+bitcount(s)dp[l][s][i]=max(dp[j][l][(i-1)])+bitcount(s) dp[l][s][i]=max(dp[j][l][(i−1)])+bitcount(s)

Code

#include<cstdio>

int dp[(1 << 10) + 5][(1 << 10) + 5][3], bin[105], bitcount[(1 << 10) + 5];
int n, m, ans;

inline int max(const int &a, const int &b) {return a > b ? a : b;}

int main() {
	scanf("%d%d", &n, &m);
	for(int i = 0; i < (1 << m); ++i) {
		int t = i;
		while(t) {
			if(t & 1) ++bitcount[i];
			t >>= 1;
		}
	}
	for(int i = 0, x; i < n; ++i) {
		for(int j = 0; j < m; ++j) {
			scanf(" %c", &x);
			if(n == 1 && m == 1 && x == 'P') {
				printf("1\n");
				return 0;
			}
			bin[i] = (bin[i] << 1) | x == 'H';
		}
	}
	for(int l = 0; l < (1 << m); ++l) {
		for(int s = 0; s < (1 << m); ++s) {
			if(!((l & (s | bin[0] | (l << 1) | (l << 2))) || (s & (bin[1] | (s << 1) | (s << 2))))) {
				dp[l][s][1] = bitcount[s] + bitcount[l];
			}
		}
	}
	for(int i = 2; i < n; ++i) {
		for(int l = 0; l < (1 << m); ++l) {
			if(!(l & (bin[i - 1] | (l << 1) | (l << 2)))) {
				for(int s = 0; s < (1 << m); ++s) {
					if(!(s & (bin[i] | (s << 1) | (s << 2) | l))) {
						for(int j = 0; j < (1 << m); ++j) {
							if(!(j & (bin[i - 2] | (j << 1) | (j << 2) | s | l))) {
								dp[l][s][i % 3] =
                                    max(dp[l][s][i % 3], dp[j][l][(i - 1) % 3] + bitcount[s]);
							}
						}
					}
				}
			}
		}
	}
	for(int l = 0; l < (1 << m); ++l) {
		for(int s = 0; s < (1 << m); ++s) {
			ans = max(ans, dp[l][s][(n - 1) % 3]);
		}
	}
	printf("%d\n", ans);
	return 0;
}

Corn Fields

Farmer John has bought a brand new rectangular pasture, divided into MMM rows and NNN columns (1≤M≤12;1≤N≤121 ≤ M ≤ 12; 1 ≤ N ≤ 121≤M≤12;1≤N≤12), where each cell is a square plot of land. John plans to plant delicious grass in some of these cells for his cows to enjoy.

Unfortunately, some land is quite barren and cannot be used to plant grass. Also, the cows like the feeling of having exclusive grass plots, so John won't choose two adjacent plots - meaning no two grass plots share a common edge.

John wants to know, without considering the total number of grass plots, how many planting schemes are available to him? (Of course, leaving the new pasture completely barren is also a valid scheme.)

Solution

I believe you've already found the pattern, right? (confident)

We define dp[i][j]dp[i][j]dp[i][j] to represent the number of schemes for the first iii rows where row iii has state jjj, and bin[i]bin[i]bin[i] represents the terrain of row iii (familiar, right?)

Same as before, for each state jjj, if a binary bit is 111, it means planting grass, otherwise not.

Still the same, let the current row state be jjj and the previous row state be kkk. Under the premise of satisfying the problem requirements, we have the transition equation:

dp[i][j]=∑dp[i−1][k]dp[i][j]=\sum dp[i-1][k] dp[i][j]=∑dp[i−1][k]

Code

#include<cstdio>

const int MOD = 100000000;
int dp[15][(1 << 12) + 5], bin[15];
int n, m, ans;

int main() {
	scanf("%d%d", &n, &m);
	for(int i = 0, x; i < n; ++i) {
		for(int j = 0; j < m; ++j) {
			scanf("%d", &x);
			bin[i] = (bin[i] << 1) | !x;
		}
	}
	for(int i = 0; i < (1 << m); ++i) {
		if(!(i & (bin[0] | (i << 1)))) {
			dp[0][i] = 1;
		}
	}
	for(int i = 1; i < n; ++i) {
		for(int j = 0; j < (1 << m); ++j) {
			if(!(j & (bin[i] | (j << 1)))) {
				for(int k = 0; k < (1 << m); ++k) {
					if(!(k & (bin[i - 1] | (k << 1) | j))) {
						dp[i][j] = (dp[i][j] + dp[i - 1][k]) % MOD;
					}
				}
			}
		}
	}
	for(int i = 0; i < (1 << m); ++i) {
		ans = (ans + dp[n - 1][i]) % MOD;
	}
	printf("%d\n", ans);
}
