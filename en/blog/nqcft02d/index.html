Vitepress Blog Link Refactoring

About 735 wordsAbout 2 min

Frontend

2025-10-10

Motivation: Hide Original Article Paths

The blog's original article paths were all plain filenames like /blog/git. We want external links to look like /blog/dsafopdf, while ensuring compatibility with static site builds, navigation, internal references, and external addresses.

Breaking down the goals:

  • Generate stable 8-character lowercase slugs and map them to original markdown files
  • Make VitePress aware of these "virtual" paths—accessible during development and generating corresponding HTML during build
  • All configurations (navigation, sidebar, featured posts, etc.) can no longer hardcode original paths
  • Actual URLs on the host must uniformly include trailing /

Generate Slugs & Configure Rewrites

First step: scan src/blog in .vitepress/config/blogSlugs.mjs, generate 8-character lowercase slugs using sha256 for all non-index.md articles, and export:

  • blogSlugMap: original filename to slug
const blogSlugMap = Object.fromEntries(
  blogEntries.map((entry) => [entry.baseName, entry.slug])
);
  • blogRewrites: blog/<file> → blog/<slug>/index.md, passed to VitePress rewrites
const blogRewrites = Object.fromEntries(
  blogEntries.map((entry) => [
    entry.relativePath,
    `blog/${entry.slug}/index.md`,
  ])
);
  • blogAliasEntries: map virtual paths back to real files in Vite's resolve.alias to ensure hot updates work properly
const blogAliasEntries = blogEntries.flatMap((entry) => {
  const replacement = path.join(SRC_DIR, entry.relativePath);
  return [
    {
      find: path.join(SRC_DIR, "blog", entry.slug, "index.md"),
      replacement,
    },
    {
      find: path.join(SRC_DIR, "blog", `${entry.slug}.md`),
      replacement,
    },
  ];
});
  • Helper functions resolveBlogLink / withTrailingSlash: convenient for reuse in config files
function resolveBlogLink(baseName) {
  const normalized = baseName.replace(/^\//, "").replace(/\.md$/, "");
  const slug = blogSlugMap[normalized];
  return `/blog/${slug ?? normalized}/`;
}

function withTrailingSlash(pathname) {
  if (!pathname) return pathname;
  return pathname.endsWith("/") ? pathname : `${pathname}/`;
}

Now when accessing /blog/kjrbhsvq/, the request gets rewritten to the actual blog/git.md, and the build outputs dist/blog/kjrbhsvq/index.html.

Update Theme and Config for Hashed Paths

Next, replace all hardcoded article links one by one:

  • Navigation (nav.js), sidebar (sidebar.js), and featured posts (featuredPosts.js) all use resolveBlogLink
  • Theme components need to use __BLOG_SLUG_MAP__ (injected via vite.define) to access slugs on the client side
const blogSlugMap =
  typeof __BLOG_SLUG_MAP__ === "undefined" ? {} : __BLOG_SLUG_MAP__;
  • transformPageData automatically adds frontmatter.permalink to each article and points relativePath to the new virtual directory to keep routing and 404 checks consistent
transformPageData(pageData) {
    const filePath = pageData.filePath;
    if (!filePath) return;

    if (filePath.startsWith("blog/")) {
        const baseName = filePath.replace(/^blog\//, "").replace(/\.md$/, "");
        if (blogSlugMap[baseName]) {
        const permalink = withTrailingSlash(`/blog/${blogSlugMap[baseName]}`);
        return {
            frontmatter: {
            ...pageData.frontmatter,
            permalink,
            },
            relativePath: `blog/${blogSlugMap[baseName]}/index.md`,
        };
        }
    }

    if (
        !filePath.endsWith("index.md") &&
        !filePath.endsWith("README.md") &&
        !pageData.frontmatter?.permalink
    ) {
        const normalized = filePath.replace(/\.md$/, "");
        const permalink = withTrailingSlash(`/${normalized}`);
        return {
        frontmatter: {
            ...pageData.frontmatter,
            permalink,
        },
        relativePath: `${normalized}/index.md`,
        };
    }
}

This also allows the license info component LicenseNotice.vue to determine if it's a regular article via page.filePath.

Canonical URL and Trailing Slashes

To avoid mixing /blog/abc and /blog/abc/, we set cleanUrls to "with-subfolders", and ensure all permalinks generated by transformPageData end with /. The final build output is index.html, preventing repeated redirects when using external links.

Rewrite Internal Markdown Links

After handling external configuration, there's one more issue: links like ./GitObject.md commonly found in article content. Since the actual output path becomes /blog/iftjzdbk/, the original relative path would result in 404.

To fix this, add a small plugin to the markdown pipeline (also in config.mjs):

  1. Traverse link_open tokens during parsing
  2. If href points to blog/*.md, *.md, or a term without /, try replacing with blogSlugMap
  3. Preserve original query parameters and hash
  4. Finally rewrite the link to /blog/<slug>/
markdown: {
    config(md) {
        md.core.ruler.after("inline", "blog-slug-links", (state) => {
        const env = state.env || {};
        const currentDir = env.relativePath
            ? path.posix.dirname(env.relativePath)
            : "";

        for (const token of state.tokens) {
            if (token.type !== "inline" || !token.children) continue;

            for (const child of token.children) {
            if (child.type !== "link_open" || !child.attrs) continue;
            const hrefAttr = child.attrs.find(([name]) => name === "href");
            if (!hrefAttr) continue;

            const rewritten = rewriteBlogHref(hrefAttr[1], currentDir);
            if (rewritten) {
                hrefAttr[1] = rewritten;
            }
            }
        }
        });
    },
}

VitePress's dead link checker can also detect this layer of rewrites, and all internal navigation is restored to normal.

LicenseNotice Fix

After replacing relativePath with the virtual directory, LicenseNotice initially couldn't get the article's true path, causing the entire block to stop displaying.

Switching to page.filePath (VitePress preserves the original markdown path during build) fixed it:

- const relative = page.value?.relativePath || ""
- if (!relative.startsWith("blog/")) return false
+ const filePath = page.value?.filePath || ""
+ if (!filePath.startsWith("blog/")) return false

Results

All articles now hide their real filenames while maintaining:

  • Theme, navigation, and internal references all use unified slugs
  • Trailing slash normalization
  • LicenseNotice, featured posts, sidebar, and other modules work properly with hashed paths
  • Markdown content doesn't require manual link rewrites
