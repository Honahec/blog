## t3

This problem seems okay, came up with an $O(Tn)$ approach:

Mark the position of another point at each point. We operate in two passes (first pass choosing L or R, using L as example): first select the first point (lexicographically) and mark it as 1, find where this number appears again, mark it as n, then find the connected component at position n (? check if there are L or R numbers on the left or right (prioritize L), mark them sequentially (constant factor is a bit large, but with $O2$ it should be fine)

As shown in the diagram:

![](https://cdn.jsdelivr.net/gh/Calm00/PicGo/img/csp2021-1.png)

Debugged for three and a half hours, three minutes before the exam ended I discovered that because the final result needs lexicographic sorting, all searches need to be written separately

Quickly copied and pasted in three minutes, didn't care about code style anymore, successfully debugged the large sample case in the last minute
